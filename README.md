# 더소피 매일학습

고3 매일학습지 20회차를 학생별로 제출·채점·누적 관리하는 웹 애플리케이션입니다.

## 구현 기능

- 학생 이름만 등록하고 개별 아이디·임시 비밀번호 발급
- 원장 계정과 선생님 공통 관리 계정 분리
- 모고·모평 답안과 변형 문제 답안을 별도로 입력하고 즉시 채점
- 문항별 정오답·정답·해설 제공
- 두 답안을 모두 제출한 뒤 영역별 괄호형 정답 전체 공개
- 학생별 20회차 누적 성적과 모고·모평/변형 평균 분리
- 교사용 제출 현황 및 학생별 성적 관리
- 학생 CSV 일괄 등록, 비밀번호 재발급, 계정 정지
- 학부모 전달용 성적표 PNG 저장 및 모바일 공유
- `고3 매일학습지1.docx`에서 20회차 정답·해설·괄호 정답 자동 추출
- SQLite 로컬 실행 / PostgreSQL 운영 배포 지원

## 로컬 실행

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# macOS/Linux: source .venv/bin/activate
pip install -r requirements.txt

# .env.example을 참고해 환경변수를 설정하거나 기본 개발값으로 실행
python app.py
```

브라우저에서 `http://127.0.0.1:5000`을 엽니다.

개발 기본 계정은 다음과 같으며 운영 배포에서는 반드시 환경변수로 바꿔야 합니다.

- 원장: `director` / `ChangeMe-Director-2026`
- 교사: `teacher` / `ChangeMe-Teacher-2026`

두 계정은 첫 로그인 후 비밀번호 변경 화면으로 이동합니다.

## 학습지 데이터 등록

1. 원장 계정으로 로그인합니다.
2. `학습지 DOCX 등록`을 엽니다.
3. 원본 `고3 매일학습지1.docx`를 업로드합니다.
4. 앱이 각 `N회차 정답 및 해설` 구역에서 다음 데이터를 추출합니다.
   - 원문 문제 정답과 해설
   - 문법·화법과 작문·문학·독서 괄호형 정답
   - 원문 문항별 변형 1·2의 정답과 해설
5. 기존 학생·제출 기록은 유지되고 회차 데이터만 갱신됩니다.

DOCX 문서의 제목 표기가 크게 바뀌면 가져오기 규칙을 조정해야 합니다. 가져오기 실패 시 원본 데이터는 변경되지 않습니다.

## 학생 CSV

```csv
name,username,password
김학생,,
```

`username`과 `password`를 비워 두면 `STU0001` 형식의 아이디와 무작위 임시 비밀번호를 발급합니다.

## 운영 배포

`render.yaml`, `Dockerfile`, `Procfile`이 포함되어 있습니다. PostgreSQL 데이터베이스와 HTTPS 웹 서비스에 배포하세요. 운영 환경에서는 다음 값을 비밀 환경변수로 지정해야 합니다.

- `SECRET_KEY`
- `DIRECTOR_PASSWORD`
- `TEACHER_PASSWORD`
- `DATABASE_URL`
- `COOKIE_SECURE=1`

GitHub Pages는 서버와 데이터베이스가 없는 정적 호스팅이므로 이 로그인 시스템을 안전하게 운영하는 용도로 사용할 수 없습니다.

## 문자·카카오톡

현재 성적표 PNG 저장과 기기의 공유 메뉴를 지원합니다. 학생이 제출하자마자 자동 문자 또는 카카오 알림톡을 보내려면 별도의 발송 업체 계정, API 키, 학부모 수신 동의 및 발송 비용이 필요합니다.


## 개인정보 최소화

학생 정보는 이름, 로그인 아이디, 암호화된 비밀번호, 학습 성적만 저장합니다. 학교, 연락처, 학부모 번호, 주소는 수집하지 않습니다.
